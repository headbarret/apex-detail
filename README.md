# APEX Detail — Лендинг

Одностраничный лендинг для студии детейлинга, тюнинга и автозвука.  
Шаблон под адаптацию для конкретного клиента.

## Стек

- React 18
- CSS (глобальный `styles.css`, без сторонних библиотек стилей)
- Intersection Observer API (плавные появления при скролле)
- Кастомный слайдер До/После (нативный drag/touch)

## Адаптация под клиента — что и где менять

### Брендинг (название, цвета)

**`Header.jsx`** и **`Footer.jsx`** — найди `APEX` и замени на название клиента.

**`styles.css`** — в самом начале блок `:root {}`, две переменные:
```css
--accent: #00E5FF;   /* основной акцент — меняй на цвет клиента */
--bg: #0B0B0B;       /* фон — можно сделать темнее/светлее */
```
Меняешь только эти две строки — цвет разойдётся по всему сайту автоматически.

### Текст на сайте

| Что | Файл | Что искать |
|---|---|---|
| Заголовок и подзаголовок hero | `Hero.jsx` | `hero-title`, `hero-subtitle` |
| Названия и описания услуг | `Services.jsx` | массив `SERVICES` в начале файла |
| Преимущества | `WhyUs.jsx` | массив `CARDS` в начале файла |
| Подпись к слайдеру | `BeforeAfter.jsx` | тег `<p className="slider-caption">` |
| Описания фото в галерее | `Gallery.jsx` | массив `PHOTOS`, поле `label` |
| Телефон | `Contacts.jsx` | `href="tel:..."` и текст рядом |
| Ссылки соцсетей | `Contacts.jsx` | массив `SOCIALS`, поля `href` и `handle` |
| Адрес и часы | `Contacts.jsx` | блок `contacts-map-placeholder` |

Все массивы с данными (`SERVICES`, `CARDS`, `PHOTOS`, `SOCIALS`) вынесены в начало своих файлов специально — редактируй только их, остальной код не трогай.

### Фотографии

Файлы кидай в `public/images/`. Пути прописаны в трёх местах:

**`Hero.jsx`** — фон:
```jsx
backgroundImage: "url('/images/hero-bg.jpg')"
```

**`BeforeAfter.jsx`** — два `<img>` подряд:
```jsx
<img src="/images/before.jpg" ... />
<img src="/images/after.jpg" ... />
```

**`Gallery.jsx`** — массив `PHOTOS`, поля `src` и `thumb` (можно одинаковые):
```jsx
{ src: "/images/gallery-1.jpg", thumb: "/images/gallery-1.jpg", label: "..." }
```

Рекомендуемые размеры: hero — 1920×1080, слайдер — 1200×675, галерея — 900×675. Сжимать до 80–85% качества (squoosh.app).

---

### Количество карточек/фото

- **Услуги** (`Services.jsx`) — можно убрать/добавить объект в массиве `SERVICES`. Сетка 2×2, если будет 3 карточки — последняя растянется на всю ширину автоматически (CSS `grid`).
- **Галерея** (`Gallery.jsx`) — массив `PHOTOS`, от 4 до 6 штук смотрится хорошо. Сетка 3 колонки, адаптируется сама.
- **Преимущества** (`WhyUs.jsx`) — массив `CARDS`, оптимально 3 штуки (сетка `repeat(3, 1fr)`).

## Особенности

- **Тёмная тема** — фон `#0B0B0B`, неоново-голубые акценты `#00E5FF`
- **Шрифты** — заголовки Montserrat, текст Inter (Google Fonts, подключены в `public/index.html`)
- **Адаптив** — брейкпоинты 600px и 900px в конце `styles.css`
- **Слайдер До/После** — кастомный, без зависимостей, работает drag + touch
- **Lightbox** — навигация стрелками клавиатуры (←/→/Esc)
- **Fade-in** — элементы с классом `.fade-in` оживают через IntersectionObserver при скролле
- **Форм нет** — только якоря и ссылки на соцсети в новом окне
